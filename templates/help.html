LAN Link Examples

Use a LAN link directly from a terminal with curl.

Basic usage

Set the link once:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'

Print the shared text:

curl -ksSL "$LAN_LINK"

Store it in a variable:

SHARED_TEXT="$(curl -ksSL "$LAN_LINK")"
printf '%s\n' "$SHARED_TEXT"

Run it as a command:

bash -c "$(curl -ksSL "$LAN_LINK")"

Using passwords

If a password is required, always send it using:

-H 'X-Access-Password: <password>'

Workspace-protected item

curl -ksSL \
  -H 'X-Access-Password: workspace-secret' \
  "$LAN_LINK"

Item-protected (Object password, overrides workspace)

curl -ksSL \
  -H 'X-Access-Password: item-secret' \
  "$LAN_LINK"

How password selection works

You only ever need one password.

LAN link access uses the most specific applicable protection rule.

DassieDrop automatically picks the right one:

  • If the item has a password, use that.
  • Otherwise, if the workspace has a password, use that.
  • Otherwise, no password is required.

Example

Workspace password: alpha

Item password: beta

Accessing the item:

Use beta

Do not use alpha

Why it works this way

LAN links are designed to be shared directly.

  • You should not need to understand the workspace structure.
  • You should not be asked for multiple passwords.

The password closest to the shared item always takes priority.

Access matrix

Workspace password Item password (Object password) What to send
No No No password required
Yes No Workspace password via X-Access-Password
No Yes Object password via X-Access-Password
Yes Yes Object password via X-Access-Password

Running scripts directly

Risky: you can pipe a LAN link into bash, but only do this if you fully trust the source and understand what will be executed.

curl -fsSL "$LAN_LINK" | bash

With a password:

curl -fsSL \
  -H "X-Access-Password: secret" \
  "$LAN_LINK" | bash

A safer approach:

curl -fsSL "$LAN_LINK" -o script.sh
less script.sh
bash script.sh

API Examples

Use the API directly from curl to share text and files.

Basic setup

Set the API host once:

API_HOST='http://your-dassiedrop-host:8000'

If the app is protected, send your automation secret as:

-H 'X-API-Key: your-api-key-or-access-code'

Share text

Share a text snippet:

curl -sS \
  -H 'Content-Type: application/json' \
  -X POST \
  -d '{
    "text": "hello from curl",
    "name": "CLI"
  }' \
  "$API_HOST/api/share-text"

Share protected text:

curl -sS \
  -H 'Content-Type: application/json' \
  -X POST \
  -d '{
    "text": "secret note",
    "name": "CLI",
    "hidden": true,
    "password": "vault"
  }' \
  "$API_HOST/api/share-text"

Share text into a specific workspace:

curl -sS \
  -H 'Content-Type: application/json' \
  -H 'X-Workspace-Name: ops-desk' \
  -X POST \
  -d '{
    "text": "hello from ops",
    "name": "CLI"
  }' \
  "$API_HOST/api/share-text"

Share text into a protected workspace:

curl -sS \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: your-api-key-or-access-code' \
  -H 'X-Workspace-Name: ops-desk' \
  -H 'X-Workspace-Password: workspace-secret' \
  -X POST \
  -d '{
    "text": "hello from protected ops",
    "name": "CLI"
  }' \
  "$API_HOST/api/share-text"

Share files

Share a file:

curl -sS \
  -X POST \
  -F 'file=@./example.txt' \
  -F 'name=CLI' \
  "$API_HOST/api/share-file"

Share a protected file:

curl -sS \
  -X POST \
  -F 'file=@./secret.pdf' \
  -F 'name=CLI' \
  -F 'hidden=true' \
  -F 'password=vault' \
  "$API_HOST/api/share-file"

Share a file into a specific workspace:

curl -sS \
  -H 'X-Workspace-Name: ops-desk' \
  -X POST \
  -F 'file=@./example.txt' \
  -F 'name=CLI' \
  "$API_HOST/api/share-file"

Share a file into a protected workspace:

curl -sS \
  -H 'X-API-Key: your-api-key-or-access-code' \
  -H 'X-Workspace-Name: ops-desk' \
  -H 'X-Workspace-Password: workspace-secret' \
  -X POST \
  -F 'file=@./example.txt' \
  -F 'name=CLI' \
  "$API_HOST/api/share-file"

Using API auth

If the app requires API auth, add X-API-Key to the same requests.

Text example with API auth

curl -sS \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: your-api-key-or-access-code' \
  -X POST \
  -d '{"text":"hello again"}' \
  "$API_HOST/api/share-text"

File example with API auth

curl -sS \
  -H 'X-API-Key: your-api-key-or-access-code' \
  -X POST \
  -F 'file=@./example.txt' \
  "$API_HOST/api/share-file"

If the workspace itself is protected, include X-Workspace-Password alongside your workspace selector.

Use the API when you want automation, workspace-aware posting, or structured JSON responses. Use LAN links when you want the simplest shareable link between devices.

General Use

The basics

DassieDrop is a local drop zone you open in a browser.

  • Open it on any device on your network
  • Share text and files instantly
  • Use it directly via the UI or via shareable LAN links

No accounts. No cloud. Everything stays on your network.

Using DassieDrop across devices

Open DassieDrop in your browser:

http://<your-ip>:8000

From there you can:

  • Paste text or upload files
  • See shared content update in real time
  • Access the same workspace from multiple devices

This is the primary way to use DassieDrop.

Workspaces

Workspaces let you separate different streams of shared text and files.

  • Each workspace has its own current text and file history.
  • The workspace button in the top-right shows your current selection.
  • Open the workspace picker to create, enter, or delete workspaces.

Creating a workspace

  1. Open the workspace picker in the top-right.
  2. Enter a workspace name.
  3. Optionally enable password protection and set a workspace password.
  4. Click Create Workspace.

Public workspaces open directly. Protected workspaces require a password when someone enters them.

Entering a workspace

  • Click the current workspace button in the top-right to open the workspace list.
  • Select a workspace row or use the Enter button.
  • If the workspace is protected, enter the workspace password and click Open.

Once entered, the main share page switches to that workspace.

Switching between devices

Workspaces are useful when multiple people or devices share the same DassieDrop server.

  • Open the same workspace on another device to see the same text and files.
  • Use different workspaces to keep separate jobs, teams, or sessions apart.

Deleting a workspace

  • Open the workspace picker and find the workspace you want to remove.
  • Click Delete.
  • If the workspace is protected, enter the workspace password to confirm deletion.

The default workspace cannot be deleted.

Sharing text

  1. Go to the Text tab.
  2. Paste your text.
  3. Click Save.
  4. Copy the LAN link or short code.

Open that link on another device to view it.

Sharing files

  1. Go to the Files tab.
  2. Drag and drop a file, or choose one manually.
  3. Copy the LAN link.

Open the link on another device to download it.

LAN links (sharing links)

A LAN link looks like:

https://<your-ip-or-host>/s/<SHORT-CODE>

This is the primary way to share content.

  • Works in any browser
  • Works across devices on your network
  • Can also be used from scripts such as curl
  • Does not require an API key

Protecting shared content

You can protect content at two levels:

1. Item (object) password

  • Use Hidden + password when sharing text or files.
  • Protects that specific item.
  • Anyone with the link must also know the password.

2. Workspace password

  • Workspaces separate content streams.
  • A protected workspace requires a password.
  • Applies to all items that do not have their own password.

Password behaviour (important)

DassieDrop keeps this simple:

  • If the item has a password, that password is required.
  • Otherwise, if the workspace has a password, that password is required.
  • Otherwise, no password is required.

You only ever need one password, never both.

Using LAN links from scripts

You can fetch shared content using curl:

curl "https://example/s/AbC123"

If a password is required:

curl "https://example/s/AbC123" \
  -H "X-Access-Password: yourPassword"

You do not need an API key for LAN links.

API access (automation)

The API is separate from LAN links.

  • Uses X-API-Key
  • Controlled by API_KEY or ACCESS_CODE
  • Intended for automation and scripting

Example:

curl -H "X-API-Key: yourKey" http://host/api/...

Downloads and previews

Routes like /download/<id> and /preview/<id> are API-style endpoints, not share links.

  • They follow API authentication rules.
  • They may require X-API-Key.

Use LAN links (/s/...) for sharing between devices.
