LAN Link Examples

Use a LAN link directly from a terminal with curl.

Set the link once and print the shared text:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
curl -ksSL "$LAN_LINK"

Store the shared text in a shell variable:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
SHARED_TEXT="$(curl -ksSL "$LAN_LINK")"
printf '%s\n' "$SHARED_TEXT"

Run the shared text as a command:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
bash -c "$(curl -ksSL "$LAN_LINK")"

If the workspace is protected, add one access password header:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
curl -ksSL \
  -H 'X-Access-Password: workspace-secret' \
  "$LAN_LINK"

If the shared item has its own password, use that same header instead. Object passwords override workspace passwords:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
curl -ksSL \
  -H 'X-Access-Password: item-secret' \
  "$LAN_LINK"

LAN link access uses the most specific applicable protection rule. Object-level password protection overrides workspace-level password protection. Workspace password protection applies only when the object does not define its own password.

LAN link access matrix:

Workspace password Object password Required access
No No No password required
Yes No Workspace password via X-Access-Password
No Yes Object password via X-Access-Password
Yes Yes Object password via X-Access-Password

General Use

How the main parts fit together.

Text tab: paste text, click Save, then copy the LAN link or short code from history.

Files tab: drag a file in or choose one, then share or download it from another device.

Hidden items: turn on Hidden and set an item password when the content itself should need a second secret.

Workspaces: use separate workspaces to keep different streams of text and files apart. A protected workspace requires a password for API access and for LAN links when the shared item does not have its own password.

API key: X-API-Key is the app-level automation secret. If API_KEY is configured, use that. Otherwise it falls back to ACCESS_CODE.

LAN links: /s/... links are the simplest way to fetch shared content from scripts. They do not use X-API-Key. If a password is needed, always send it as X-Access-Password.

Password precedence on LAN links: object password first, workspace password second, no password if neither is set.

Downloads and previews: direct file routes like /download/<id> and /preview/<id> are still API-style routes and keep their own header rules.
