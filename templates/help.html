LAN Link Examples

Use a LAN link directly from a terminal with curl.

LAN link access uses the most specific applicable protection rule. Object-level password protection overrides workspace-level password protection. Workspace password protection applies only when the object does not define its own password.

LAN link access matrix:

Workspace password Object password Required access
No No No password required
Yes No Workspace password via X-Access-Password
No Yes Object password via X-Access-Password
Yes Yes Object password via X-Access-Password

Set the link once and print the shared text:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
curl -ksSL "$LAN_LINK"

Store the shared text in a shell variable:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
SHARED_TEXT="$(curl -ksSL "$LAN_LINK")"
printf '%s\n' "$SHARED_TEXT"

Run the shared text as a command:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
bash -c "$(curl -ksSL "$LAN_LINK")"

If the workspace is protected, add one access password header:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
curl -ksSL \
  -H 'X-Access-Password: workspace-secret' \
  "$LAN_LINK"

If the shared item has its own password, use that same header instead. Object passwords override workspace passwords:

LAN_LINK='https://your-dassiedrop-host:8443/s/TASD'
curl -ksSL \
  -H 'X-Access-Password: item-secret' \
  "$LAN_LINK"

Risky: if you are intentionally sharing a script and want to execute it directly, you can pipe the LAN link into bash. Only do this if you fully trust the source and have checked what you are about to run.

curl -fsSL https://your-dassiedrop-host:8443 | bash

If the shared item is protected:

curl -fsSL https://your-dassiedrop-host:8443 \
  -H "X-Access-Password: secret" | bash

Be sure you know what you are running before you use this pattern.

General Use

The basics

DassieDrop is a local drop zone you open in a browser.

  • Open it on any device on your network
  • Share text and files instantly
  • Use it directly via the UI or via shareable LAN links

No accounts. No cloud. Everything stays on your network.

Using DassieDrop across devices

Open DassieDrop in your browser:

http://<your-ip>:8000

From there you can:

  • Paste text or upload files
  • See shared content update in real time
  • Access the same workspace from multiple devices

This is the primary way to use DassieDrop.

Sharing text

  1. Go to the Text tab.
  2. Paste your text.
  3. Click Save.
  4. Copy the LAN link or short code.

Open that link on another device to view it.

Sharing files

  1. Go to the Files tab.
  2. Drag and drop a file, or choose one manually.
  3. Copy the LAN link.

Open the link on another device to download it.

LAN links (sharing links)

A LAN link looks like:

https://<your-ip-or-host>/s/<SHORT-CODE>

This is the primary way to share content.

  • Works in any browser
  • Works across devices on your network
  • Can also be used from scripts such as curl
  • Does not require an API key

Protecting shared content

You can protect content at two levels:

1. Item (object) password

  • Use Hidden + password when sharing text or files.
  • Protects that specific item.
  • Anyone with the link must also know the password.

2. Workspace password

  • Workspaces separate content streams.
  • A protected workspace requires a password.
  • Applies to all items that do not have their own password.

Password behaviour (important)

DassieDrop keeps this simple:

  • If the item has a password, that password is required.
  • Otherwise, if the workspace has a password, that password is required.
  • Otherwise, no password is required.

You only ever need one password, never both.

Using LAN links from scripts

You can fetch shared content using curl:

curl "https://example/s/AbC123"

If a password is required:

curl "https://example/s/AbC123" \
  -H "X-Access-Password: yourPassword"

You do not need an API key for LAN links.

API access (automation)

The API is separate from LAN links.

  • Uses X-API-Key
  • Controlled by API_KEY or ACCESS_CODE
  • Intended for automation and scripting

Example:

curl -H "X-API-Key: yourKey" http://host/api/...

Downloads and previews

Routes like /download/<id> and /preview/<id> are API-style endpoints, not share links.

  • They follow API authentication rules.
  • They may require X-API-Key.

Use LAN links (/s/...) for sharing between devices.
